# 暑假前后端实习学习路线（24周版）

下面给你一条「从小白 → 暑假能投前端/后端实习」的可执行学习路线（按暑假倒推，约 24 周）。你提到“自己搭建网站”，**非常建议**：能把项目做出来、部署上线、写清楚 README 和技术点，基本就是简历里最硬的东西。

---

## 你要达到的“实习可用”标准是什么

很多前端/全栈实习 JD 会写得很像：  
- 会 **HTML/CSS/JavaScript**，理解 ES6、异步（Promise / async-await）、代码规范（ESLint）等。  
- 至少掌握一个主流框架（**React 或 Vue**），最好搭过完整页面 + 数据请求 + 路由 + 状态管理。  
- 如果是后端/全栈：能写 **REST API**、连 **MySQL** 做 CRUD、懂基本鉴权（JWT / Session）、会基本部署更好。  

换句话说，你的目标不是“把课看完”，而是做到：
1) **能独立完成一个可用的 Web 项目（前端 + 后端 + 数据库）**  
2) **能部署上线（别人能打开链接使用）**  
3) **能讲清楚你怎么设计、怎么排查问题、怎么取舍**（面试最爱问）

---

## 先选一个主线：别两边都学成“浅尝辄止”

你想“前后端相关”，我建议你走 **前端主线 + 后端补齐到能做 API** 的路线：更容易在 3~4 个月内做出作品，投递时也更像“能上手干活”。

### 推荐技术栈（按性价比排序）

**方案 A（最适合新手、最快做出全栈作品）**  
- 前端：HTML/CSS/JS → TypeScript → React（或 Vue）  
- 后端：Node.js → Express / Nest（二选一）  
- 数据库：MySQL（必学）+ Redis（加分）  
- 工程化：Git、ESLint、Docker（部署时非常加分）

**方案 B（如果你更想投偏后端的岗位）**  
- 后端：Java → Spring Boot → MySQL → Redis  
- 前端：会用 Vue/React 做基本页面即可  

你现在是小白，我默认你走 **方案 A**（更快闭环）。

---

## 24 周学习路线（每周 12~18 小时左右：每天 2~3h + 周末半天）

### 第 1–2 周：环境 + 基本功打底（必须快）
**目标：能写静态页面 + 用 Git 管代码**
- 装好：VS Code、Node.js、Git  
- 学会：Git 基本操作（clone / commit / branch / PR 思维）  
- HTML：语义化、表单、常见标签  
- CSS：盒模型、flex、grid、响应式、BFC（不用背概念，能用出来）  
- JS：变量、函数、数组对象、作用域、DOM、事件

**产出：**  
- 一个个人介绍页（响应式，手机电脑都能看）  
- 上传 GitHub：有 README、截图

---

### 第 3–5 周：JavaScript 核心 + 浏览器与网络
**目标：你写的 JS 不再“靠蒙”**
- JS 必会：原型链/this（理解即可）、闭包、模块化、事件循环（宏/微任务要能解释）  
- 异步：Promise、async/await、错误处理  
- 浏览器：DevTools 调试、Network 面板看请求  
- HTTP 基础：请求方法、状态码、Cookie、缓存、CORS

**产出：**  
- 一个“带搜索 + 分页”的小应用（调用公开 API，比如天气/电影/图书），包含 loading、错误态、空数据态

---

### 第 6–8 周：TypeScript + 工程化（让你像“正规开发”）
你要把“写得出来”升级成“写得像团队代码”。

- TypeScript：类型、接口、泛型、联合类型、类型收窄（会写 React 组件类型即可）  
- 工程化：Vite、ESLint + Prettier、环境变量、目录结构  
- 会用一个 UI 组件库：Ant Design（React）/ Element Plus（Vue）二选一

**产出：**  
- 把前面的小应用重构成 TS 项目  
- README 写清楚：技术栈、功能、如何启动、项目结构

---

### 第 9–12 周：框架主线（React 或 Vue 二选一深挖）
你不需要“精通所有框架”，但要把一个吃透。

**React 路线（示例）**
- 组件、Hooks、受控组件、表单  
- 路由（React Router）  
- 状态管理：优先学 **React Query / SWR**（请求状态管理）+ 少量全局状态（Zustand/Redux 选一个轻的）  
- 性能与体验：列表渲染、组件拆分、懒加载、节流防抖

**产出：管理后台前端（纯前端版先跑起来）**
- 登录页（先假登录/Mock）  
- 列表页：增删改查、分页、筛选、弹窗表单  
- 统一封装 request（axios/fetch）+ 拦截器 + 错误提示

---

### 第 13–16 周：后端入门到“能写 API”（全栈的分水岭）
**目标：让你的管理后台“接上真实数据”**
- Node.js：模块、npm、基本调试  
- Express / Nest：路由、Controller、Service、Middleware  
- API 设计：REST 风格、分页、参数校验、统一返回结构  
- 鉴权：JWT（access/refresh 的基础思路）  
- MySQL：表设计、主键/索引、常见查询、事务概念  
- ORM（二选一）：Prisma（更新手友好）/ TypeORM  
- 文件上传（可选加分）：头像/附件上传

**产出：**  
- 你的后台项目从“假数据”升级为“真后端 + 真数据库”  
- 最少做 8 个接口：登录、用户信息、列表分页、详情、创建、编辑、删除、上传

---

### 第 17–20 周：把项目做“像实习项目”，补齐加分项
**目标：让你的项目能扛住面试追问**
- 代码质量：分层、错误处理、日志（console→logger）  
- 缓存（加分）：Redis 用在验证码/热点数据/会话（简单场景即可）  
- 测试（加分但别死磕）：接口用 Postman/Apifox，写少量单元测试（Jest）  
- 安全常识：密码加密（bcrypt）、SQL 注入/参数校验、CORS 配置

**产出：**  
- README 增加：接口文档、数据库 ER 图（哪怕手画也行）、部署说明、你做过的优化点

---

### 第 21–24 周：部署上线 + 简历/面试冲刺
**部署上线是“实习竞争力放大器”**
- 前端部署：Vercel / Netlify 或云服务器 Nginx  
- 后端部署：云服务器 + Nginx 反代 + PM2/Docker  
- HTTPS：Let’s Encrypt（会配是加分项）  
- CI/CD（加分）：GitHub Actions 自动部署（不强求，但很好看）

**面试准备（每天一点，别最后堆）**
- 算法：保持手感（数组、哈希、双指针、二分、BFS/DFS）  
- 八股：HTTP、浏览器缓存、跨域、事件循环、MySQL 索引/事务、JWT 流程

---

## 你至少要做 2 个“可展示项目”（建议照这个规格做）

### 项目 1：个人作品集网站（1–2 周完成，持续迭代）
**必须有：**  
- 你的简介、技术栈、项目列表、项目链接（线上地址 + GitHub）  
- 响应式、加载优化（图片压缩/懒加载）、SEO 基本项  
- 部署上线（一个可访问 URL）

> 这个项目让面试官“点得开你的人”。

### 项目 2：全栈管理系统（核心项目，8–12 周打磨）
**建议选题（任选其一）：**
- 校园二手交易平台（商品发布/搜索/收藏/消息）  
- 学习打卡/笔记系统（标签、全文检索可选）  
- 社团/活动管理（报名、审核、名单导出）

**必须实现的“实习级功能清单”：**
- 登录/注册（或统一登录）+ 权限（普通用户/管理员）  
- 列表分页 + 条件筛选 + 排序  
- CRUD（创建/编辑/删除）  
- 文件上传（头像/附件）  
- 后端：参数校验、统一错误处理、日志  
- 数据库：至少 6 张表，关系清晰（1对多、多对多出现一次更好）  
- 部署上线：前端 + 后端 + 数据库（哪怕数据库不公网暴露也行）

**你要准备好回答：**
- 你为什么这么设计表？索引怎么加？  
- 分页怎么做？性能怎么考虑？  
- 鉴权怎么做？token 过期怎么办？  
- 遇到过什么 bug？怎么定位的？

## 学习资源怎么选（避免资源地狱）
优先用：官方文档 + 1 门系统课 + 大量动手做项目（比例大概 3:7）。

---

## 每天怎么学最有效（给你一个模板）
- 30–45 分钟：看一小节内容（不要连续看 2 小时）  
- 60–90 分钟：照着目标功能敲出来（今天就产出可运行结果）  
- 15 分钟：写学习笔记/踩坑记录（面试复盘材料）  
- 30 分钟：刷 1 题算法（保持手感）

---

## 最容易踩的坑（提前避开）
1) **学太多框架**：React/Vue 只精一个就够，另一个“了解”放到后面。  
2) **只做页面不做数据闭环**：一定要接 API + 数据库，否则很像“课程作业”。  
3) **项目不上线**：没有链接=面试官很难快速相信你做过。  
4) **简历写技术名词堆砌**：要写“你做了什么、解决了什么问题、有什么结果”。

---

## 最短闭环目标（建议）
- **6 周内：作品集网站 + 一个前端小应用（带真实 API）**  
- **12 周内：全栈 CRUD 系统并上线**
